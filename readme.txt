项目说明文档：北京西城区城市交通噪声交互可视化与治理方案模拟

我这个项目希望回答的核心问题是：在北京西城区这种典型的车多人多高密度城市环境中，交通噪声在不同道路和不同时段如何分布，如果对某一条路采取限速、装隔音屏障绿化带、铺降噪路面等措施，噪声会发生什么样的变化。围绕“哪里噪声干扰多、什么时候严重、为什么严重、怎么降噪”四个方面，我们可以在地图中点击任一道路，查看这条路在当前时段下的噪声强度，并通过参数调节时时观察地图整体热力分布与局部的指标变化，从而对城市交通噪声以及治理策略有一个直观的理解。

在设计上，我选择地图作为主视图，因为噪声它是具有空间属性的，通过地图我们可以对道路和建筑位置有直观清晰的认知。交互方面，支持自由点击选择道路和切换时段，选择道路后可以对该段路进行限速调整，安装隔音屏障还有铺设降噪沥青。可视化模式可以选择热力图和等值面两种。热力图用来展示整体噪声分布，一眼就能看出哪里吵哪里安静。等值线模式，可以更精细地看某个阈值以上的区域。我还给道路加了流动动画，车速越快粒子流动越快，晚上车少的时候粒子密度也会降低，这样不用看数字也能感觉到“车多吵、车少安静”。右边数据面板和选中道路对应展示对比“没采取措施”和“采取措施”的效果。然后替代方案的话，有考虑过比较复杂的声学仿真，这个虽然理论会更精确，但需要更高质量的交通流量、建筑反射等很多参数，还有更高的计算开销，在相应的时间难以完成，最后选择了这种简化了的模型，虽然没有办法达到环境评价级别的精确度，但有合理的噪声变化趋势，也能保证有一个比较好的交互体验。

外部资源方面，道路数据来源于 OpenStreetMap ，地图底图来自高德地图；核心可视化与交互技术栈包括 Leaflet、D3、Vue等。

整个开发大概花了20个小时左右，先是选定主题，明确思路，然后准备数据，接着做可视化，交互，修复bug，优化效果。我觉得比较花时间的是对数据的准备处理还有具体的细节任务的实现问题的解决。比如最开始用的是几根线条代表道路效果并不是很好，还有出现了如坐标系不一致导致道路偏移、图层叠在一起点不中道路、热力图和等值线切换时状态不一致等问题。

项目满足静态部署要求，不依赖后端，可以直接托管在GitLab Pages或GitHub Pages上运行。
